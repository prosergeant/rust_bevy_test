Создай **структурированный план разработки 2D игры на Rust с Bevy (ECS)**, используя **аналогичную архитектуру и стиль плана**, как в предоставленном примере `update1_plan.md`. План должен быть достаточно подробным для разработки, но **не слишком раздутым**, чтобы на каждом шаге было понятно, что и когда делать, без глубоких подробностей кода.

Требования к структуре:
1. **Обзор архитектуры проекта**
   - Перечислить основные плагины (каждый отдельный функциональный блок)
   - Описать ключевые компоненты, ресурсы и основные системы
   - Указать как и в каком порядке плагины регистрируются в `main.rs`

2. **Приоритеты реализации**
   - Разбить функциональность на 2–3 уровня приоритетов (например: быстрые победы, заметные улучшения, расширение контента)
   - Для каждого уровня описать:
     * Цель/назначение фичи
     * Какие плагины/модули нужны
     * Какие новые компоненты/ресурсы/системы добавить

3. **Технические шаги для каждого пункта**
   - Для каждого шага указать конкретные действия, например:
     * Создать новый плагин
     * Добавить ресурс
     * Реализовать систему
   - **Не углубляться до строк кода**, достаточно названия сущностей и их роли

4. **Структура файлов проекта**
   - Как должна выглядеть организация каталогов/файлов по функциональным модулям

5. **Метрики качества (опционально)**
   - Краткие целевые метрики (производительность, тесты), аналогичные примеру

6. **Ограничения на детализацию**
   - План должен быть **до 3 уровней вложенности**
   - Для каждого пункта ясно указать **цель и ожидаемый результат**
   - Не включать подробную реализацию API или конкретные строки кода

Формат ответа:
- Нумерация логическая: 1., 1.1, 1.2, …
- Краткие описания, но структурированные
- Акцент на **архитектуру, модули/плагины, компоненты, системы**, а не на внутреннюю имплементацию
